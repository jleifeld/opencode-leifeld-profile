# OpenCode Self-Learning System Implementation Plan

This file turns `OPENCODE_SELF_LEARNING_PLAN_V2.md` into an execution-ready checklist.

The tasks below are deliberately small. Each task should take about 2-5 minutes, has exact file paths, exact code or patch content, and concrete verification steps.

These instructions assume the repo is still in the current state I inspected:

- `opencode.jsonc` has only the existing `AGENTS.md` instruction
- `AGENTS.md` contains only the current OpenCode rules block
- `.gitignore` currently contains only `!.opencode`
- `.opencode/package.json` already contains `@opencode-ai/plugin`

No `package.json` change is needed for this implementation plan because the plugin code and tests only use:

- `@opencode-ai/plugin`
- Node built-ins
- Bun's built-in TypeScript test runner

## Execution Rules

1. Apply the tasks in order.
2. Use the exact patches below.
3. Run the verification for each task before moving on.
4. After the code tasks are done, stop and ask the user whether `.opencode/learnings/.stats.json` should be committed or ignored.

## Automated Test Strategy

This plan includes real automated coverage, not just manual smoke checks.

Use `bun test .opencode/tests` as the automated test command.

Automated coverage in this plan:

- plugin-level integration tests for `learning_export_session`
- plugin hook tests for the `read`-based hit counter
- static contract tests for `opencode.jsonc`, `AGENTS.md`, the command files, and the agent file

Manual coverage remains necessary only for the model-driven and TUI-specific parts:

- slash command registration in the TUI
- child-session startup
- the interactive review loop with `y/e/n/m`
- full end-to-end quality of extracted learnings

Definition of done:

- `bun test .opencode/tests` passes
- manual TUI smoke tests pass

## Task 7: Create the `learning-extractor` subagent

Estimated time: 5 minutes

### Files

- `.opencode/agents/learning-extractor.md`

### Exact patch

```diff
*** Begin Patch
*** Add File: .opencode/agents/learning-extractor.md
+---
+description: Extracts reusable learnings from the current session export and saves approved ones under .opencode/learnings/
+mode: subagent
+hidden: true
+temperature: 0.2
+permission:
+  bash: deny
+  webfetch: deny
+  edit:
+    "*": deny
+    ".opencode/learnings/**": allow
+---
+
+You extract durable, project-specific learnings from the current session and save user-approved ones under `.opencode/learnings/`.
+
+## Required workflow
+
+1. Call `learning_export_session` first unless the command already gave you an export path.
+2. Read `.opencode/learnings/INDEX.md` before proposing new learnings.
+3. Read the exported session markdown file.
+4. Extract `0-5` strong candidates.
+5. Review candidates one at a time.
+6. Save only explicitly approved learnings.
+7. Regenerate `.opencode/learnings/INDEX.md` from `.opencode/learnings/items/*.md` after saving.
+
+## What qualifies as a learning
+
+Good candidates:
+
+- non-obvious root causes
+- project-specific config or tooling gotchas
+- repeated failed attempts before the working fix
+- durable workarounds that will save time later
+- user clarifications with clear reuse value
+
+Reject these:
+
+- generic best practices
+- standard framework usage
+- one-off bug fixes with no reuse value
+- typo corrections and session noise
+
+## Review loop
+
+Use this exact format and wait after every candidate:
+
+```text
+-----------------------------------------------
+Candidate [N/M]: <Title>
+Category: <category>   Tags: [a, b, c]
+
+Problem:   <1-3 sentences>
+Cause:     <1-3 sentences>
+Solution:  <concrete fix>
+-----------------------------------------------
+[y] accept   [e] edit   [n] reject   [m <slug>] merge
+```
+
+Handle replies exactly:
+
+- `y` -> queue the candidate for saving
+- `e` -> ask what should change, revise the same candidate, and present it again
+- `n` -> discard it and continue
+- `m <slug>` -> merge into `.opencode/learnings/items/<slug>.md`
+
+Do not move to the next candidate until the current one is resolved.
+
+## Saved learning file format
+
+Path:
+
+`.opencode/learnings/items/<slug>.md`
+
+Use this structure:
+
+```markdown
+---
+title: <human-readable title>
+slug: <kebab-case slug>
+category: auth | build | framework | tooling | data | infra | ui | testing | other
+tags: [kebab-case, keywords]
+discovered: YYYY-MM-DD
+source_session: <session-id>
+source_export: exports/YYYY-MM-DD_<session-id>.md
+---
+
+## Problem
+<1-3 sentences>
+
+## Root Cause
+<1-3 sentences>
+
+## Solution
+<specific fix>
+
+## Related
+- <optional related slug>
+```
+
+## Index format
+
+Regenerate `.opencode/learnings/INDEX.md` from scratch.
+
+Use this structure:
+
+```markdown
+# Project Learnings Index
+
+> This file is loaded automatically via `opencode.jsonc` instructions.
+> Read a detailed learning file only when the current task clearly matches it.
+
+<!-- Auto-generated by /learn. Manual edits may be overwritten. -->
+
+## Tooling & CLI
+- `items/example.md` - One-line summary.
+```
+
+Category mapping:
+
+- `auth` -> `## Auth & Security`
+- `build` -> `## Build & Deploy`
+- `framework` -> `## Framework Quirks`
+- `tooling` -> `## Tooling & CLI`
+- `data` -> `## Data & Persistence`
+- `infra` -> `## Infrastructure & Deploy`
+- `ui` -> `## UI & Styling`
+- `testing` -> `## Testing`
+- `other` -> `## Other`
+
+## Path safety
+
+You may write only inside `.opencode/learnings/`.
+Never edit source code, docs outside the learning area, or `.opencode/learnings/.stats.json`.
+
+## Empty or unreadable session export
+
+If the export is empty, missing, or clearly unusable, say so plainly and stop without writing files.
*** End Patch
```

Estimated time: 5 minutes

### Files

### Exact patch

```diff
*** Begin Patch
*** Add File: .opencode/plugins/learning-system.ts
+import { tool, type Plugin } from "@opencode-ai/plugin"
+import { existsSync } from "node:fs"
+import { mkdir, readFile, writeFile } from "node:fs/promises"
+import path from "node:path"
+
+const LEARNINGS_DIR = ".opencode/learnings"
+const EXPORTS_DIR = `${LEARNINGS_DIR}/exports`
+const ITEMS_DIR = `${LEARNINGS_DIR}/items`
+const STATS_FILE = `${LEARNINGS_DIR}/.stats.json`
+const MAX_TOOL_OUTPUT_CHARS = 1200
+
+type LearningStats = Record<
+  string,
+  {
+    hits: number
+    first_used: string | null
+    last_used: string | null
+  }
+>
+
+function toAbsoluteProjectPath(directory: string, filePath: string) {
+  return path.isAbsolute(filePath) ? filePath : path.resolve(directory, filePath)
+}
+
+function toProjectRelativePath(directory: string, filePath: string) {
+  return path.relative(directory, toAbsoluteProjectPath(directory, filePath)).split(path.sep).join("/")
+}
+
+function getTrackedLearningSlug(directory: string, filePath: string | undefined) {
+  if (!filePath) return null
+
+  const relativePath = toProjectRelativePath(directory, filePath)
+  if (!relativePath.startsWith(`${ITEMS_DIR}/`)) return null
+  if (!relativePath.endsWith(".md")) return null
+
+  const baseName = path.basename(relativePath)
+  if (baseName.startsWith(".")) return null
+
+  return baseName.replace(/\.md$/, "")
+}
+
+function truncate(text: string, max = MAX_TOOL_OUTPUT_CHARS) {
+  if (text.length <= max) return text
+  return `${text.slice(0, max).trimEnd()}\n[output truncated]`
+}
+
+function redactSensitiveText(text: string) {
+  return text
+    .replace(/Authorization:\s*[^\n]+/gi, "Authorization: [redacted]")
+    .replace(/\b(?:ghp_|github_pat_|sk-)[A-Za-z0-9_-]+\b/g, "[redacted]")
+}
+
+function summarizeToolOutput(text: string) {
+  const trimmed = String(text ?? "").trim()
+  if (!trimmed) return "[no output]"
+  return truncate(redactSensitiveText(trimmed))
+}
+
+function formatTimestamp(value: number | undefined) {
+  if (!value) return "unknown"
+  return new Date(value).toISOString()
+}
+
+function formatToolPart(part: any) {
+  const state = part.state
+
+  if (state.status === "completed") {
+    return [
+      `### Tool: ${part.tool}`,
+      `Status: completed`,
+      `Input: ${JSON.stringify(state.input)}`,
+      `Output: ${summarizeToolOutput(state.output)}`,
+    ].join("\n")
+  }
+
+  if (state.status === "error") {
+    return [
+      `### Tool: ${part.tool}`,
+      `Status: error`,
+      `Input: ${JSON.stringify(state.input)}`,
+      `Error: ${summarizeToolOutput(state.error)}`,
+    ].join("\n")
+  }
+
+  if (state.status === "running") {
+    return [
+      `### Tool: ${part.tool}`,
+      `Status: running`,
+      `Input: ${JSON.stringify(state.input)}`,
+    ].join("\n")
+  }
+
+  return [
+    `### Tool: ${part.tool}`,
+    `Status: pending`,
+    `Input: ${JSON.stringify(state.input)}`,
+  ].join("\n")
+}
+
+function formatPart(part: any): string | null {
+  if (part.type === "text") {
+    const text = String(part.text ?? "").trim()
+    return text || null
+  }
+
+  if (part.type === "tool") return formatToolPart(part)
+
+  if (part.type === "subtask") {
+    return [
+      `### Subtask: ${part.agent}`,
+      `Description: ${part.description}`,
+      part.command ? `Command: ${part.command}` : null,
+    ]
+      .filter(Boolean)
+      .join("\n")
+  }
+
+  if (part.type === "agent") {
+    return `### Agent\n${part.name}`
+  }
+
+  return null
+}
+
+function formatMessage(entry: any) {
+  const role = entry.info.role === "assistant" ? "Assistant" : "User"
+  const renderedParts = entry.parts.map(formatPart).filter(Boolean)
+
+  return [
+    `## ${role}`,
+    `- Message ID: ${entry.info.id}`,
+    `- Created: ${formatTimestamp(entry.info.time?.created)}`,
+    "",
+    renderedParts.length > 0 ? renderedParts.join("\n\n") : "[no exported parts]",
+  ].join("\n")
+}
+
+async function ensureLearningDirs(directory: string) {
+  await mkdir(path.join(directory, EXPORTS_DIR), { recursive: true })
+  await mkdir(path.join(directory, ITEMS_DIR), { recursive: true })
+}
+
+async function loadStats(directory: string): Promise<LearningStats> {
+  const absolutePath = path.join(directory, STATS_FILE)
+  if (!existsSync(absolutePath)) return {}
+
+  try {
+    const raw = await readFile(absolutePath, "utf-8")
+    return JSON.parse(raw) as LearningStats
+  } catch {
+    return {}
+  }
+}
+
+async function saveStats(directory: string, stats: LearningStats) {
+  await mkdir(path.join(directory, LEARNINGS_DIR), { recursive: true })
+  await writeFile(path.join(directory, STATS_FILE), `${JSON.stringify(stats, null, 2)}\n`, "utf-8")
+}
+
+async function exportSessionMarkdown(client: any, directory: string, sessionID: string) {
+  const sessionResult = await client.session.get(
+    { sessionID, directory },
+    { throwOnError: false },
+  )
+
+  if (sessionResult.error || !sessionResult.data) {
+    throw new Error(`Could not load session ${sessionID}`)
+  }
+
+  const messagesResult = await client.session.messages(
+    { sessionID, directory, limit: 500 },
+    { throwOnError: false },
+  )
+
+  if (messagesResult.error || !messagesResult.data) {
+    throw new Error(`Could not load messages for session ${sessionID}`)
+  }
+
+  const exportedAt = new Date().toISOString()
+  const exportDate = exportedAt.slice(0, 10)
+  const relativePath = `${EXPORTS_DIR}/${exportDate}_${sessionID}.md`
+  const absolutePath = path.join(directory, relativePath)
+
+  const markdown = [
+    "# Session Export",
+    "",
+    `- Session ID: ${sessionID}`,
+    `- Title: ${sessionResult.data.title || "Untitled"}`,
+    `- Exported At: ${exportedAt}`,
+    `- Export Type: current-effective-session-state`,
+    "",
+    "## Messages",
+    "",
+    ...messagesResult.data.map(formatMessage),
+    "",
+  ].join("\n")
+
+  await writeFile(absolutePath, markdown, "utf-8")
+
+  return {
+    relativePath,
+    messageCount: messagesResult.data.length,
+  }
+}
+
+export const LearningSystem: Plugin = async ({ client, directory }) => {
+  void directory
+
+  return {
+    tool: {
+      learning_export_session: tool({
+        description: "Export the current session to a markdown file under .opencode/learnings/exports/",
+        args: {
+          sessionID: tool.schema.string().optional().describe("Session ID to export. Defaults to the current session."),
+        },
+        async execute(args, context) {
+          const sessionID = args.sessionID ?? context.sessionID
+
+          await ensureLearningDirs(context.directory)
+          const exported = await exportSessionMarkdown(client, context.directory, sessionID)
+
+          return `Saved session export to ${exported.relativePath} (${exported.messageCount} messages)`
+        },
+      }),
+    },
+  }
+}
*** End Patch
```
